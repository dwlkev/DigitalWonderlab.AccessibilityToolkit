What do visual checks currently cover?

Visual checks analyse computed color contrast on the rendered page. They complement the 37 server-side HTML checks by catching issues that only appear after CSS is applied.

Why might screenshot previews be missing?

Some pages block capture due to browser security constraints (for example cross-origin assets). The report still includes selectors and contrast details for every issue found.

How does licensing work?

Core functionality is free forever. When no licence key is configured, all features are enabled for development and evaluation. See the Settings tab for licence details.

Does this guarantee WCAG AA compliance?

This toolkit helps you work toward WCAG AA, but automated scans cannot catch every issue and do not replace expert review. If you need deeper coverage, visit the Help & Services tab.

How can I report issues or request features?

Use the GitHub issues tracker: Accessibility Toolkit Issues

What WCAG levels can I test against?

You can choose Level A, AA, or AAA. Level AA is the default and the most commonly required standard. Higher levels include all checks from lower levels.

Can I exclude certain pages or document types from audits?

Yes. Go to the Settings tab to exclude specific document types or individual pages. Excluded items are skipped during site audits.
